# v1.0.3 — 로그인 창 stale 쿠키 버그 수정

릴리즈 일자: 2026-04-20
대상 브랜치: `release/v1.0.3` → `main`

---

## 문제 (Symptom)

> "로그인 버튼을 누르면 탭이 뜨다가 사라짐. 일부 사용자에서만 재현."

- 로그인 버튼 클릭 → Claude 로그인 창이 잠깐 뜨고 즉시 닫힘
- 닫힌 직후 사용량 데이터는 여전히 가져오지 못함
- 무한 루프: 로그인 버튼이 다시 표시됨

## 원인 (Root Cause)

`electron/electron_main.js` 의 `ipcMain.handle('login')` 구현에 두 가지 결함이 있었습니다.

### 1. 영속 partition + 즉시 캡처 조합

```js
const loginWin = new BrowserWindow({
  webPreferences: { partition: 'persist:claude-login', ... }
});
loginWin.webContents.on('did-navigate', tryCapture);  // 첫 navigation에서 발동
```

- `persist:claude-login` partition은 앱 재시작 후에도 쿠키를 보존
- 과거 로그인 시도/세션 만료로 남아있던 stale 쿠키가 그대로 남아있음
- 새 로그인 창이 `claude.ai/login` 로딩 시 첫 `did-navigate` 이벤트에서 `tryCapture` 발동
- stale 쿠키가 형식 검사(`sk-ant-sid` 접두 + `lastActiveOrg` 존재)를 통과 → 즉시 창 닫힘

### 2. 검증 부재

```js
if (sessionKey && orgCookie) {
  // 쿠키 존재만으로 성공 판정 → 만료된 쿠키도 통과
  loginWin.close();
}
```

- 쿠키의 *존재*만 체크하고 실제 인증된 세션인지 검증하지 않음
- 이후 `fetchUsage`가 401로 실패해도 이미 창은 닫힌 후

### 왜 일부 사용자만 재현되는가

- **재현 사용자**: 과거에 로그인 창을 띄웠다가 만료된 stale 쿠키가 partition에 남아있음
- **재현 안 됨**: partition이 비어있거나, 마지막 세션이 아직 유효함

## 수정 사항 (Changes)

### `electron/electron_main.js:335-453`

#### ① partition 쿠키 사전 클리어
```js
const ses = session.fromPartition('persist:claude-login');
await ses.clearStorageData({ storages: ['cookies'] });
```
로그인 창을 띄우기 전에 partition의 쿠키를 모두 지워 stale 쿠키 false-positive 차단.

#### ② 캡처 후 API 검증 단계 추가
```js
function validateSession(orgId) {
  // net.request로 https://claude.ai/api/organizations 호출
  // 200~299 응답일 때만 valid 판정
}

if (sessionKey && orgCookie) {
  const valid = await validateSession(orgCookie.value);
  if (!valid) { /* 검증 실패 카운트 증가, 창 유지 */ return; }
  // 200 OK일 때만 storage 저장 + 창 닫기
}
```
쿠키 존재 → API 200 응답까지 확인한 뒤에만 인증 완료로 간주.

#### ③ 초기 navigation 무시 (debounce)
```js
let pageReady = false;
loginWin.webContents.once('did-finish-load', () => {
  setTimeout(() => { pageReady = true; }, 1000);
});

async function tryCapture() {
  if (handled || !pageReady) return;
  // ...
}
```
`loadURL` 직후 첫 `did-navigate`에서 오발동하지 않도록 1초 지연 후부터 캡처 활성화.

#### ④ 5분 타임아웃 + 명확한 실패 메시지
```js
const timeout = setTimeout(() => {
  if (!handled) {
    loginWin.close();
    resolve({ ok: false, error: '로그인 대기 시간 초과 (5분)' });
  }
}, LOGIN_TIMEOUT_MS);
```
검증 5회 연속 실패 시 `'쿠키 검증 실패. 다시 로그인해 주세요.'` 메시지 표시.
